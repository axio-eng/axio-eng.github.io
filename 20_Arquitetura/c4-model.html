Modelo de Arquitetura C4 para NEXUS ​

Este documento descreve a arquitetura do sistema NEXUS utilizando o modelo C4, incorporando as decisões arquiteturais registradas nos ADRs ativos. O NEXUS é uma arquitetura de memória externa para LLMs e agentes, combinando proximidade vetorial (T3/Qdrant), estrutura relacional (T2/PostgreSQL) e navegação topológica (T4 Multi-Relational Graph) para expandir o contexto além do top_k flat.

Nível 1: Diagrama de Contexto do Sistema ​

O diagrama de contexto do sistema NEXUS ilustra como o sistema interage com seus usuários e sistemas externos.

Principais Interações:

  • Consulta (Query Flow): LLMs/Agentes enviam consultas ao NEXUS. O NEXUS embeda a query, busca sementes no T3 (Qdrant), expande o contexto no T4 (PostgreSQL) e hidrata o conteúdo do T2 (PostgreSQL) para materializar packs coerentes.
  • Pipeline de Enriquecimento: O NEXUS ingere dados do File System (T1), extrai AST (L1), gera abstrações semânticas (L2) e embeddings, e constrói o grafo T4.
  • Operações Admin: Desenvolvedores e MCP Clients podem acionar operações administrativas como enriquecimento (nexus_admin(action='enrich')).

Nível 2: Diagrama de Contêiner ​

O diagrama de contêiner detalha os principais serviços e armazenamentos de dados que compõem o sistema NEXUS.

ADRs e Contêineres:

  • Coordinator (Rust): É o coração do sistema, onde a maioria das decisões arquiteturais se manifesta.
    • ADR-0001 (Propagação de Carga): Afeta a lógica de atualização de pesos de arestas e propagação de sinais no grafo T4, executada no Coordinator.
    • ADR-0002 (Operadores Relacionais Composáveis): Define os operadores T4 (project, abstract, compile, chain) implementados no Coordinator.
    • ADR-0003 (Nomad Workers): Descreve a decomposição dos loops de background do Coordinator em Nomad traits.
    • ADR-0021 (Forgetful Swarm Dynamics): A lógica de decay_score e detecção de Conflict Islands é implementada no Coordinator.
    • ADR-0023 (Pipeline Evolution): O AST-Aware Chunking e a métrica tokens·hora são implementados no pipeline de enriquecimento do Coordinator.
    • ADR-0024 (Backpropagation Charges): A propagação de heat e o decay são gerenciados no Coordinator (query.rs, decay.rs).
    • ADR-0025 (Tipagem Vetorial por Projeção): As ideias de coleções Qdrant por projeção e dimensionalidade variável afetariam o roteamento de embeddings no Coordinator.
  • Nomad Workers: São os loops de background que executam as tarefas de enriquecimento e manutenção.
    • ADR-0003 (Nomad Workers): Define a estrutura e o comportamento dos Nomad Workers.
    • ADR-0021 (Forgetful Swarm Dynamics): O DecayRoutine é um Nomad Worker.
    • ADR-0023 (Pipeline Evolution): Os loops L1/L2 são Nomad Workers, e o AST-Aware Chunking e Context GC são implementados neles.
    • ADR-0024 (Backpropagation Charges): O DecayNomad é responsável pelo resfriamento do heat.
  • PostgreSQL (T2): Armazena o estado persistente do grafo e das abstrações.
    • ADR-0001 (Propagação de Carga): Armazena os pesos das arestas e cargas potenciais.
    • ADR-0002 (Operadores Relacionais Composáveis): Armazena as entidades topológicas (subgrafos rotulados).
    • ADR-0021 (Forgetful Swarm Dynamics): Adiciona decay_score à tabela links.
    • ADR-0023 (Pipeline Evolution): Armazena os chunks gerados pelo AST-Aware Chunking.
    • ADR-0024 (Backpropagation Charges): Adiciona a coluna heat à tabela abstractions.
    • ADR-0025 (Tipagem Vetorial por Projeção): As ideias de tabelas relacionais por nó afetariam o esquema do PostgreSQL.
  • Qdrant (T3): Armazena os embeddings vetoriais.
    • ADR-0001 (Propagação de Carga): Utilizado para a busca semântica (T3) que alimenta a retropropagação.
    • ADR-0025 (Tipagem Vetorial por Projeção): As ideias de coleções Qdrant por projeção e dimensionalidade variável afetam diretamente o Qdrant.

Nível 3: Diagrama de Componentes ​

O diagrama de componentes detalha os módulos internos do Coordinator e a interação entre eles.

ADRs e Componentes:

  • Query Handler:
    • ADR-0001 (Propagação de Carga): A lógica de retropropagação de feedback e cargas dipolo é integrada ao processamento de query.
    • ADR-0002 (Operadores Relacionais Composáveis): Os operadores T4 (project, abstract, compile, chain) são invocados pelo Query Handler.
    • ADR-0024 (Backpropagation Charges): O heat é alimentado no query.rs após uma consulta bem-sucedida.
  • Enrichment Pipeline:
    • ADR-0023 (Pipeline Evolution): Implementa o AST-Aware Chunking e a instrumentação para a métrica tokens·hora.
  • T4 Graph Runtime:
    • ADR-0001 (Propagação de Carga): Gerencia os pesos das arestas e a propagação de sinais GSP.
    • ADR-0002 (Operadores Relacionais Composáveis): Contém a implementação dos operadores canônicos T4.
    • ADR-0021 (Forgetful Swarm Dynamics): Gerencia o decay_score e a detecção de Conflict Islands.
    • ADR-0024 (Backpropagation Charges): Gerencia a coluna heat nas abstrações e seu decaimento.
    • ADR-0025 (Tipagem Vetorial por Projeção): Roteia as operações do grafo para coleções Qdrant específicas por projeção.
  • Nomad Registry & Nomad Workers:
    • ADR-0003 (Nomad Workers): Define a estrutura do Nomad trait e a orquestração dos workers.
    • ADR-0021 (Forgetful Swarm Dynamics): O DecayNomad é um worker que aplica o protocolo de esquecimento.
    • ADR-0023 (Pipeline Evolution): Os workers L1 e L2 são afetados pelo AST-Aware Chunking e Context GC.
    • ADR-0024 (Backpropagation Charges): O DecayNomad executa o resfriamento do heat.
  • GSP Module:
    • ADR-0001 (Propagação de Carga): Contém a lógica para Graph Signal Processing, cargas dipolo, operadores de translado/dilatação e aproximação de Chebyshev.
  • Vector Typing Module:
    • ADR-0025 (Tipagem Vetorial por Projeção): Implementa a lógica para criar e gerenciar coleções Qdrant por projeção, com dimensionalidade variável.

Nível 4: Diagrama de Código (Abstrações Chave) ​

Este nível foca em abstrações de código específicas e suas relações com os ADRs.

ADRs e Abstrações de Código:

  • ADR-0001 (Propagação de Carga): A Edge Weight Update Formula e os conceitos de cargas dipolo e aproximação de Chebyshev são as abstrações centrais.
  • ADR-0002 (Operadores Relacionais Composáveis): As T4 Operator Signatures definem a interface para a manipulação de entidades topológicas.
  • ADR-0003 (Nomad Workers): O Nomad Trait é a abstração fundamental para a decomposição dos loops de background.
  • ADR-0021 (Forgetful Swarm Dynamics): O Decay Score Mechanism e a lógica de Conflict Islands são as abstrações chave.
  • ADR-0023 (Pipeline Evolution): O Context GC Algorithm com AST-Aware Chunking e tokens·hora são as abstrações de código.
  • ADR-0024 (Backpropagation Charges): A Heat Update Logic para a coluna heat e seu decaimento são as abstrações.
  • ADR-0025 (Tipagem Vetorial por Projeção): A Vector Typing Interface para gerenciar coleções Qdrant por projeção é a abstração.

Nota sobre Discrepância do ADR-0001 ​

O ADR-0001 apresenta uma discrepância entre suas fontes:

  • O cabeçalho do arquivo (docs/adr/0001-backpropagation-abstractions-charges.md) indica "Proposto (Proposed)"
  • O índice geral (docs/adr/index.md) o marca como ✅ Active
  • O índice local (docs/adr/README.md) o marca como "Proposto"

Para fins deste modelo C4, o ADR-0001 é tratado como 📝 Proposto (alignado com o cabeçalho do documento), uma vez que a implementação Chebyshev/GSP/dipolo ainda não foi realizada — o que existe é o heat propagation (ADR-0024), que é uma simplificação da proposta original.

Nota sobre ADR-0025 ​

O ADR-0025 está em estágio 💭 Pensamento/Discussão. Nenhuma decisão foi tomada. As ideias incluem:

  • Múltiplas coleções Qdrant por projeção T4
  • Classificador LLM para tipo semântico (em vez de extensão de arquivo)
  • Dimensionalidade variável por tipo de dado
  • Tabelas relacionais separadas no PostgreSQL

Este ADR não deve ser tratado como aceito ou implementado.
