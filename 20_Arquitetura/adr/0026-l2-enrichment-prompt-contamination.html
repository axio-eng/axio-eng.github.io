ADR-0026: L2 Enrichment — Prompt Contamination Causa Alucinação Sistêmica ​

Status: Aceito
Date: 2026-06-27
Deciders: @ops

Context ​

O pipeline de enriquecimento L2 (services/coordinator/src/enrich.rs) gera resumos semânticos (L2) a partir da AST estrutural (L1) de cada arquivo. O L2 alimenta a camada de busca semântica (T3/Qdrant) e a navegação topológica (T4).

Durante uma avaliação pós-hoc de taxa de alucinação, descobriu-se que ~82% dos L2s continham referências a funções, triggers e tabelas que não existem no arquivo descrito, mas existem em arquivos vizinhos no mesmo diretório ou em arquivos com AST similar.

Diagnóstico ​

O Prompt Problemático (enrich.rs:707-723) ​

O prompt de geração L2 concatena três blocos de contexto:

AST Structure (L1 do próprio arquivo)
+ [MODULE: migrations] (perfil de TODOS os arquivos no mesmo diretório)
+ [SIMILAR ASTs] (arquivos com AST similar via busca vetorial Qdrant)

O modelo não recebe nenhuma instrução para ignorar entidades dos contextos auxiliares. Ele interpreta o contexto do módulo e as ASTs similares como conteúdo pertinente ao arquivo atual e os incorpora no resumo.

Cadeia Causal ​

Arquivo: seed_super_admin_iam.sql (2 INSERTs, 0 funções)
         │
         ▼
L1 (ast-grep): corretamente vazio
         │
         ▼
Fase 1b — Module Context: agrega 50+ arquivos do diretório migrations/
  → inclui create_product_with_stock, get_clinic_ui_config, etc.
         │
         ▼
Fase 1a — RAG Context: busca ASTs similares
  → retorna mais arquivos com create_product_with_stock
         │
         ▼
Modelo recebe: "AST: {}" + [MODULE: migrations (create_product_with_stock...)]
  + [SIMILAR ASTs (sim=0.92: product_stock.sql)]
         │
         ▼
L2 gerado: "Função create_product_with_stock cria produtos..."
  → ALUCINAÇÃO: função não existe neste arquivo

Evidência Quantitativa ​

MétricaValor
L2 examinados28
L2 com funções alucinadas23 (82%)
Função mais alucinadacreate_product_with_stock (15+ ocorrências)
Top 5 alucinadascreate_product_with_stock, add_appointment, add_stock_movement, get_clinic_ui_config, perform_procedure
Origem das funçõesArquivos SQL vizinhos no diretório migrations/

O L1 (ast-grep) estava correto em 100% dos casos — o erro é exclusivamente do prompt de enriquecimento, não da extração estrutural.

Impacto ​

  1. Busca semântica (T3) corrompida: L2s contaminados poluem os embeddings no Qdrant, fazendo com que queries semanticamente relacionadas retornem chunks de arquivos incorretos.
  2. Navegação topológica (T4) ruidosa: Arestas semantic_bridge computadas a partir de L2s contaminados ligam nós que não têm relação real.
  3. Métrica de alucinação (hallucination_metrics): a coluna existe no schema PostgreSQL mas nunca é populada pelo pipeline — não há detecção automatizada.

Decision Drivers ​

  • Correção factual é requisito não-negociável para um sistema de memória operacional para LLMs
  • L2 contaminado propaga erro para T3 (embeddings) e T4 (arestas semânticas)
  • L1 (ast-grep) é determinístico e confiável — serve como âncora de verdade
  • Module context é útil para o MACRO mas perigoso se mal isolado

Considered Options ​

Option A: Remover module_ctx e rag do prompt L2 ​

Remover completamente os contextos auxiliares do prompt de geração L2.

  • Pros: Elimina a contaminação na raiz; L2 descreve apenas o arquivo atual
  • Cons: MACRO analysis perde contexto do módulo; modelo não sabe como o arquivo se relaciona com a vizinhança

Option B: Instrução negativa explícita + separação de seções ​

Manter os contextos mas com:

  1. Instrução explícita: "The MODULE and SIMILAR ASTs below are reference only. Do NOT include functions, entities, or behaviors from those contexts in your MICRO analysis. Only describe what is present in THIS file's AST Structure."
  2. Seção "Reference" separada visualmente após a instrução
  • Pros: Mantém contexto útil pro MACRO; abordagem cirúrgica
  • Cons: Depende do modelo respeitar a instrução; requer validação empírica

Option C: Pós-processamento com verificador L1 ​

Gerar L2 sem contexto auxiliar e depois enriquecer o MACRO separadamente com o contexto do módulo.

  • Pros: Isolamento completo entre MICRO (factual) e MACRO (contextual)
  • Cons: Duas chamadas de LLM por arquivo; maior latência e custo

Decision ​

Adotar Option A — remover module_ctx e rag do prompt L2 completamente.

Racional ​

Option B (instrução negativa explícita) foi testada primeiro e reduziu a taxa de contaminação de 37% para 32% (n=25) — melhora de apenas 5pp. A instrução negativa é insuficiente porque o modelo não possui um mecanismo intrínseco para filtrar entidades de contexto auxiliar.

Option A foi adotada como solução definitiva: o prompt L2 contém apenas a AST estrutural (L1) do próprio arquivo, sem module_ctx ou RAG asts.

Prompt Corrigido (Option A) ​

rust
let prompt = format!(
    "Analyze the following code unit's L1 AST structure (static analysis from ast-grep) and produce:\n\
     \n\
     MICRO — What does this specific unit (function, class, module, or file) do? \
     Describe its purpose, core logic, inputs/outputs, and key operations in 1-3 plain sentences. \
     Focus on intent and behavior, not structure.\n\
     \n\
     MACRO — How does this unit fit into the larger module? \
     What does it depend on, what depends on it, and what architectural role does it play? 1-3 sentences.\n\
     \n---\n\
     AST Structure (THIS FILE ONLY):\n{}",
    row.content
);

Consequences ​

Positive ​

  • L2 factual accuracy deve subir para próximo de 1.0 (contaminação eliminada na raiz)
  • Embeddings no Qdrant passam a representar o conteúdo real de cada arquivo
  • Arestas semantic_bridge no T4 tornam-se confiáveis

Negative ​

  • MACRO analysis perde contexto de vizinhança — o modelo não sabe como o arquivo se relaciona com irmãos no mesmo módulo
  • Mitigação futura: MACRO pode ser enriquecido em passo separado (pós-L2) com contexto do módulo, similar à Option C

Risks ​

  • A remoção de module_ctx pode reduzir a qualidade do MACRO em arquivos muito pequenos (ex: arquivos de 1 função sem contexto de módulo)
  • Mitigação: Monitorar L2s gerados e reavaliar via eval.py

Related Decisions ​

  • ADR-0017 (legacy): L2 Structure Inheritance from L1 — estabeleceu que L2 deve herdar estrutura do L1, mas não antecipou contaminação por contexto
  • ADR-0018 (legacy): Hallucination Detection and MCP Refinement — abordagem de detecção pós-hoc vs. prevenção na fonte (esta ADR)

Implementation Notes ​

  • Arquivo: services/coordinator/src/enrich.rs linha 707-723 (Option A aplicada)
  • Modelo usado: qwen2.5-coder:nexus via Ollama
  • Enriquecimento rodou em 3 batches: 2026-06-25 (30 L2s), 2026-06-27 (43 L2s), 2026-06-27 (reingestão com Option A)
  • Baseline contaminação: 37% (n=73, prompt com module_ctx + rag)
  • Option B (instrução negativa): 32% (n=25) — insuficiente
  • Option A (remoção total): 0% contaminação (n=41, validação empírica em 2026-06-27)
  • hallucination_metrics no schema PostgreSQL deve ser populada após a correção

Teorema da Projeção Horizontal no Grafo part_of ​

Formulação Formal ​

Seja (T) uma árvore de diretórios (hierarquia part_of do T4). Seja (P) um nó pai (módulo) com filhos (F_1 \ldots F_n) (arquivos). Seja (L2(F_i)) o resumo semântico gerado para o filho (i). Seja (\text{propriedades}(F_j)) o conjunto de funções/entidades definidas no arquivo (j).

Teorema: Se (L2(F_i)) é gerado a partir de um prompt que inclui (\bigcup_{j=1}^{n} \text{propriedades}(F_j)) (agregação de todos os irmãos), então (P(L2(F_i) \text{ contém elemento de } F_j \mid j \neq i) \to 1) à medida que (n) cresce. A projeção horizontal de propriedades agregadas de irmãos sobre um nó individual é assintoticamente certa.

Esboço de Prova ​

  1. O prompt inclui module_ctx = agregação de TODOS os arquivos no diretório
  2. O modelo não recebe mecanismo para distinguir "esta entidade é de um arquivo irmão" de "esta entidade é do arquivo atual"
  3. O treinamento do modelo (next-token prediction) otimiza para texto coerente, não para filtragem de contexto espúrio
  4. Portanto, entidades do agregado são probabilisticamente projetadas sobre o L2 do nó atual
  5. A probabilidade cresce com (n) porque mais entidades irmãs competem pela atenção do modelo no contexto agregado

Validação Empírica ​

ConfiguraçãoTaxa de Contaminaçãon
Baseline (module_ctx + rag)37%73
Option B (instrução negativa)32%25
Option A (remoção total)0% 🟢41

Corolário para o T4 ​

Arestas part_of no grafo T4 não implicam herança de propriedades. O modelo de grafos separa corretamente estrutura (part_of) de semântica (semantic_bridge), mas o pipeline de enriquecimento L2 estava conflitando-os ao projetar propriedades agregadas sobre nós individuais.

Implicações: Arestas semantic_bridge computadas a partir de L2 pré-Option A são parcialmente não confiáveis. Módulos com L2 ativo antes da correção devem ser reindexados com L2 limpos.
