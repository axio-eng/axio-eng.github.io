Parte III: Setup, Testes e Contribuição ​

Setup do Ambiente ​

bash
# 1. Pré-requisitos
sudo dnf install docker podman docker-compose
sudo systemctl start docker

# 2. Subir stack de dados (bancos apenas — sem Ollama)
docker compose -f docker-compose.standalone.yml up -d postgres qdrant

# 3. Verificar saúde
curl -s http://localhost:6333/collections | jq '.result.collections[] | .name'
curl -s http://localhost:5432  # deve responder

# 4. Build do coordinator
cd services/coordinator
cargo build --release

Boas Práticas de Contribuição ​

  1. Sempre use nexus_guide() primeiro ao iniciar trabalho no repositório.
  2. Nunca use grep/find/cat loops — as MCP tools existem para isso.
  3. Documentação é saída, não entrada — markdown tem nexus: ingestible: false.
  4. Commit messages: tipo(escopo): verbo no imperativo — motivo.
  5. Antes de mudar arquitetura, leia os ADRs em docs/adr/.
  6. Para reindexar: nexus_admin(action='enrich').
