Glossário NEXUS ​

40+ termos essenciais para navegar o sistema.

A ​

Abstraction : Unidade de conhecimento em um nível específico (L0-L4). Armazenada na tabela abstractions do PostgreSQL. Cada abstraction tem um blob_id (arquivo de origem), level (nível de abstração), chunk_index (posição no blob) e content (conteúdo textual).

ADR (Architecture Decision Record) : Documento que registra uma decisão arquitetural significativa, incluindo contexto, opções consideradas, decisão tomada e consequências. Os ADRs do NEXUS estão em docs/adr/.

Agent (Agente de IA) : Consumidor primário do NEXUS. Um LLM que navega no grafo via MCP tools para entender uma base de código.

Agent Links : Tabela no PostgreSQL (agent_links) que armazena as arestas do grafo multi-relacional T4. Contém source_id, target_id, criteria (tipo da aresta), weight e created_at.

ast-grep : Ferramenta de análise sintática estática usada para extrair estruturas de código (funções, classes, interfaces) de arquivos fonte. É o motor do L1.

B ​

Blob : Arquivo fonte completo ingerido pelo sistema. Armazenado na tabela blobs com fs_path, source_hash, raw_content e metadata. Um blob é dividido em múltiplas abstractions (chunks).

C ​

Chunk : Fragmento de um blob em um nível de abstração. Um chunk L0 é um trecho de código fonte; um chunk L1 é uma estrutura AST extraída; um chunk L2 é uma descrição semântica gerada por LLM.

Context Pack : Grupo de abstrações que formam uma janela de contexto coerente para o agente. Inclui uma abstração central + chunks vizinhos do mesmo blob. Materializado pelo build_context_packs() em query.rs.

Context Event : Entrada de observabilidade no pipeline. Registra toda operação que consome tokens (prompts, respostas, embeddings) para auditoria e garbage collection. Tabela context_events.

Contradicts : Tipo de aresta T4 que conecta abstrações semanticamente conflitantes. Ex: duas versões da mesma função com comportamentos diferentes.

D ​

Decaimento (Decay) : Mecanismo proposto (ADR-0021) para reduzir gradualmente o peso de arestas obsoletas no grafo T4, permitindo que conexões irrelevantes "esfriem" e desapareçam.

Dependências (depends_on:imports) : Tipo de aresta T4 que conecta um arquivo a outro que ele importa. Representa o esqueleto estrutural do código.

Deriva Semântica (Semantic Drift) : Perda de precisão semântica ao longo do tempo ou entre saltos de contexto. O NEXUS mitiga isso combinando T3 (busca aproximada) com T4 (navegação exata).

E ​

Embedding : Vetor numérico (768d no NEXUS) que representa o significado semântico de um texto. Gerado por nomic-embed-text.

Enrichment Pipeline : Pipeline que transforma código bruto (L0) em abstrações estruturadas (L1), semânticas (L2) e links topológicos (T4). Executado por run_enrichment() em loop contínuo.

F ​

File Node : Nó no grafo T4 que representa um arquivo completo (L4). Todos os chunks L0/L1/L2 de um arquivo se conectam ao seu file node via part_of. É o ponto de entrada estrutural para navegação.

Filesystem Watch : Mecanismo que monitora mudanças no sistema de arquivos (create, modify, delete) e dispara re-ingestão automática. Implementado via notify crate.

G ​

Grafo Multi-Relacional (T4) : Estrutura de dados que conecta abstrações através de 4 tipos de aresta (part_of, depends_on:imports, semantic_bridge, contradicts) com 3 eixos de projeção (criteria, file_type, project).

GPS (metáfora T3) : A camada de busca vetorial (Qdrant) que entende intenção e proximidade semântica. "Onde fica a lógica de agendamento?"

H ​

Heat (Calor) : Métrica de atividade/importância no grafo T4. Abstrações acessadas frequentemente recebem mais heat; o heat decai com o tempo. Implementação proposta no ADR-0024.

I ​

Ingest Queue : Máquina de estados para controle de ingestão de arquivos. Estados: pending → processing → completed | failed. Tabela ingest_queue.

L ​

L0 (Nível 0) : Chunk bruto de código fonte. O nível mais baixo de abstração.

L1 (Nível 1) : Abstração estrutural. Extraída deterministicamente via ast-grep do arquivo completo. Contém funções, classes, interfaces, structs — o que o código é.

L2 (Nível 2) : Abstração semântica. Gerada por LLM (Ollama) usando contexto L1 + RAG. Contém descrições em linguagem natural do propósito do código — o que o código faz.

L3 (Nível 3) : Síntese arquitetural. Emerge da detecção de comunidades no grafo T4, não de um blob individual. Descreve o propósito arquitetural de um módulo inteiro.

L4 (Nível 4) : File node. Representa um arquivo completo no grafo T4. Não é uma abstração do conteúdo, mas um nó estrutural para navegação.

M ​

Mapa da Cidade (metáfora T4) : A camada de grafo (PostgreSQL) que representa a estrutura exata do código: CEP (caminho do arquivo), hierarquia (quem contém quem), imports (quem depende de quem).

MCP (Model Context Protocol) : Protocolo padronizado para comunicação entre LLMs e ferramentas. O NEXUS expõe suas capacidades como tools MCP via servidor SSE em localhost:7474.

Module : Agrupamento semântico de arquivos dentro de um projeto. Classificado por regras em nexus_config.json (module_rules).

N ​

nexus_adjacent() : Tool MCP que caminha no grafo T4. Dado um node_id, retorna os vizinhos com suas arestas. A navegação pode ser filtrada por criteria (tipo de aresta).

nexus_admin() : Tool MCP para administração do sistema. Ações: enrich (força pipeline), ingest (clona repositório), status (progresso de ingestão).

nexus_guide() : Tool MCP que retorna o protocolo operacional completo do NEXUS. Deve ser chamada primeiro por qualquer agente ao iniciar trabalho.

nexus_link() : Tool MCP que cristaliza uma descoberta do agente como uma nova aresta no grafo T4 (memória stigmergic). Permite que um agente deixe trilhas para agentes futuros.

nexus_map() : Tool MCP que retorna o mapa estrutural de um projeto ou módulo: lista de arquivos, módulos e métricas de deriva arquitetural.

nexus_query() : Tool MCP principal para busca. Combina T3 (busca semântica) com triangulação L1 (estrutura) e retorna Context Packs. Parâmetros: query, extensions, collection, triangulate, top_k, max_chunks_per_file, context_radius.

nexus_reflections() : Tool MCP que retorna insights arquiteturais de alto nível (L3) gerados pelo sistema.

nomic-embed-text : Modelo de embeddings da Ollama usado pelo NEXUS. Gera vetores 768d para busca semântica. Executa em CPU. Latência ~85ms.

P ​

PPR (Personalized PageRank) : Algoritmo de walk no grafo T4 usado para expandir contexto a partir de seeds semânticas. Propaga scores pelas arestas, gerando um subgrafo rankeado.

Pipeline Metrics : Métricas temporais coletadas a cada chamada ao Ollama: TTFT, TPOT, throughput, prompt eval count. Tabela pipeline_metrics.

Projection : Fatia do grafo T4 definida pela interseção de 3 eixos: criteria × file_type × project. Toda navegação é uma projeção.

Q ​

qwen2.5-coder:nexus : Modelo de linguagem local usado para geração de descrições semânticas L2 e síntese L3. 1.5B parâmetros, 29 layers offloaded na GPU. ~4.5GB VRAM.

S ​

Semaphore : Mecanismo de controle de concorrência. O NEXUS usa um semáforo global de 2 permits para chamadas ao Ollama, prevenindo OOM na GPU.

semantic_bridge : Tipo de aresta T4 que conecta arquivos semanticamente similares. Representa a "carne" do grafo — conexões que emergem de similaridade de propósito.

Stigmergy : Comunicação indireta entre agentes através do ambiente. No NEXUS, agentes registram descobertas via nexus_link(), criando arestas que agentes futuros podem navegar.

Superseded By : Mecanismo de encadeamento temporal (T6). Uma abstração pode ser "substituída por" outra quando o arquivo fonte é re-ingerido, formando uma cadeia de versões.

T ​

T1 : Camada de Filesystem. Fonte da verdade original.

T2 : Camada de PostgreSQL. Âncora canônica de dados estruturados.

T3 : Camada de Qdrant. Busca vetorial por proximidade semântica.

T4 : Camada de Grafo Multi-Relacional. Navegação topológica com arestas tipadas.

T5 : Camada de Reflections. Insights arquiteturais emergentes (L3).

T6 : Camada de Time. Encadeamento temporal de versões.

Tokio : Runtime assíncrono Rust usado pelo coordinator. Gerencia os loops de enrichment, file watcher, query server e MCP server concorrentemente.

Triangulação : Cruzamento de resultado semântico (L2) com estrutura (L1) para validação. Quando triangulate=true em nexus_query(), cada resultado L2 é verificado contra sua contraparte L1.

W ​

Weaver : Componente do pipeline que cria arestas T4. Chama process_batch() no final de cada ciclo L2 para gerar links part_of, depends_on:imports, semantic_bridge e contradicts.
