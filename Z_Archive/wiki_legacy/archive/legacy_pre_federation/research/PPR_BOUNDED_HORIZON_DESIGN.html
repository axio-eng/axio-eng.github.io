Horizonte Bounded de PPR: Por Que 3 Hops e Não Mais ​

Status: Decisão de arquitetura profunda
Data: 2026-03-24
Documentos companheiros: BOUNDED_DIFFUSION_T4.md, GEOMETRIA_DE_FLUXO_T3.md, FORMAL_ANALYSIS_HAMILTONIAN.md

Propósito ​

Este documento articula uma escolha de design que é facilmente mal interpretada como limitação. Não é.

O sistema NEXUS executa exatamente 3 hops de Personalized PageRank (PPR) sobre o grafo T4 e para. A maioria dos sistemas de navegação topológica empírica usaria "mais hops" ou "até convergência" como default. O NEXUS deliberadamente não faz isso.

A razão é matemática, não pragmática. E a diferença entre as duas abordagens é qualitativa, não apenas de grau.

1. A armadilha da distribuição estacionária ​

O que acontece se deixarmos o PPR convergir ​

Se executarmos PPR com teleportação até o equilíbrio, obtemos:

s* = lim_{k→∞} P^k · s_0

Onde P é a matriz de transição estocástica do grafo e s_0 é a distribuição inicial (sementes do Qdrant).

Em um grafo não-direcionado, a distribuição estacionária π tem uma propriedade simples e devastadora:

π_i ∝ degree_i

O PageRank converge para uma distribuição proporcional ao grau do nó. Isso é um resultado da teoria de cadeias de Markov, não uma aproximação.

Em um grafo de código ou documentos, onde nós altamente conectados são "utils", "manager", "config", "index" e similares, a distribuição estacionária não retorna "contexto semanticamente relevante para a query". Retorna "os nós mais conectados do grafo".

O NEXUS evita isso completamente. Não por tunagem, mas por estrutura.

A formulação correta ​

O sistema NEXUS computa:

s_3 = P³ · s_0

Exatamente 3 potências da matriz de transição. Não converge. Não itera até tolerância. Para.

Isso é um horizonte limitado (bounded horizon), não uma versão truncada de convergência. A diferença é fundamental.

2. Prova informal de por que grau domina a distribuição estacionária ​

Considere uma aresta (i, j) em um grafo não-direcionado. A probabilidade de transição de i para j é:

P(j|i) = w(i,j) / Σ_k w(i,k)

Em um grafo não-ponderado, w(i,j) = 1 para todas as arestas, então:

P(j|i) = 1 / degree(i)

O Random Walk salta de i para cada vizinho com probabilidade igual 1/degree(i). Nós com alto grau são visitados mais frequentemente porque têm mais arestas incidentes que conduzem a eles.

Em equilíbrio, a equação de balanceamento detalhado (detailed balance) exige:

π_i · P(j|i) = π_j · P(i|j)

Substituindo as expressões acima para grafo não-ponderado:

π_i / degree(i) = π_j / degree(j)

Portanto, para qualquer par de nós conectados:

π_i · degree(j) = π_j · degree(i)

Como isso vale para todos os pares em uma componente conectada, segue que:

π_i ∝ degree(i)

QED. A distribuição estacionária é proporcional ao grau para grafos não-direcionados.

No contexto do NEXUS, isso significa que se o sistema executasse PPR até equilíbrio, os nós retornados seriam ordenados principalmente por conectividade estrutural, não por relevância semântica para a query. O T4 seria dominado pela topologia, não pela semântica.

3. A matemática do horizonte limitado: s_3 vs s* ​

Distribuição após 3 hops ​

s_3 = P³ · s_0

Cada entrada (s_3)_i é a soma de todos os caminhos de comprimento exatamente 3 que terminam em i, ponderados pela condutância de cada aresta e pelo score inicial.

Explicitamente:

(s_3)_i = Σ_{j,k} P_{ij} · P_{jk} · P_{ki} · (s_0)_k

Isso é o somatório sobre todos os caminhos de 3 arestas de qualquer seed k até o nó i. A ordem importa: estamos computando paths, não relaxando para um equilíbrio.

Distribuição no equilíbrio ​

s* = lim_{k→∞} P^k · s_0 = π

Para grafo não-direcionado, π_i = degree(i) / (2·|E|). A distribuição inicial s_0 é completamente diluída. A query desaparece. O que sobra é puramente estrutural.

A interpretação qualitativa ​

Propriedades_3 = P³ · s_0s* = lim P^k · s_0

O horizonte de 3 hops preserva a locality da query. O equilíbrio a destrói.

4. A interpretação do cone de luz ​

Considere a difusão de score como um cone de luz emanando das sementes.

No tempo 0, o cone está concentrado nas sementes Qdrant. No tempo 1 (primeiro hop), ele se expande para os vizinhos imediatos. No tempo 2, para os vizinhos dos vizinhos. No tempo 3, para o terceiro nível.

Após 3 hops, o cone parou. Ele não atingiu todo o grafo. Ele não convergiu para o equilíbrio. Ele iluminou uma vizinhança semântica ao redor das sementes.

A analogia física é precisa: assim como a luz de uma estrela a 3 anos-luz não ilumina galáxias distantes, o score PPR a 3 hops não captura a estrutura global do grafo. Ele captura o vizinhário imediato no espaço topológico.

Se deixássemos o cone se expandir indefinidamente (equilíbrio), ele preencheria todo o grafo com um brilho uniforme, indistinguível da radiação cósmica de fundo. A localização da estrela original seria irrelevante.

O NEXUS deliberadamente para no terceiro hop para preservar a direção do cone. A luz que chega a um nó depois de 3 saltos ainda carrega informação sobre a estrela de origem. Depois de mais hops, a informação se degrada para ruído estrutural.

Por que 3? ​

Três não é arbitrário. É o ponto onde:

α³ ≈ 0.85³ ≈ 0.614

O fator de teleportação α = 0.85 reduz o score em 15% a cada hop por reinjeção direta nas sementes. Após 3 hops, o score acumulado de qualquer caminho é menor que PPR_MIN_SCORE = 0.01. Qualquer nó além do terceiro hop recebe menos de 1% do score original — abaixo do limiar de ruído (main.rs:23-25).

O código no main.rs:812 implementa isso diretamente:

rust
const PPR_ALPHA: f64 = 0.85;
const PPR_MAX_HOPS: usize = 3;
const PPR_MIN_SCORE: f64 = 0.01;

for _hop in 0..PPR_MAX_HOPS {
    if scores.is_empty() { break; }
    // ... propagação de score
    next.retain(|_, v| *v &gt;= PPR_MIN_SCORE);
}

Três hops é o ponto de corte onde o custo de continuar (ruído topológico) supera o benefício (mais cobertura). É uma escolha determinada pela distribuição de score, não por intuição.

5. A intuição geodésica: caminho mais curto, não walk aleatório ​

PageRank standard é um random walk. O nó final de um random walk longo é determinado por grau, não por direção.

O PPR do NEXUS com 3 hops é melhor entendido como propagação de caminho geodésico. Cada hop não é um passo de random walk — é um salto ao longo da geodésica no espaço semântico.

A distinção importa:

Random Walk (equilíbrio):

  • Pergunta: "Onde o random walker termina depois de infinitos passos?"
  • Resposta: "Nos nós de maior grau."
  • Isso é útil para pagerank global, ranking de importância estrutural.

Caminhada Geodésica (horizonte limitado):

  • Pergunta: "Quais nós são alcançáveis por caminhos curtos e semanticamente condutores a partir das sementes?"
  • Resposta: "Os nós na vizinhança topológica das sementes."
  • Isso é útil para montagem de contexto: você quer saber o que está semanticamente próximo, não o que é estruturalmente central.

O NEXUS não está fazendo random walk. Está fazendo navegação geodésica com 3 saltos permitidos. A limitação de hops não é uma aproximação de random walk — é a definição de um regime onde geodésica e random walk divergem e apenas geodésica importa.

A GEOMETRIA_DE_FLUXO_T3.md articula isso em termos de campo e trajetória. O runtime do T4 (main.rs:812-868) o implementa como propagação de score por matriz de adjacência com horizonte fixo.

O documento BOUNDED_DIFFUSION_T4.md explica os três mecanismos ortogonais que previnem explosão combinatória. Este documento explica por que parar é o objetivo, não o constrangimento.

6. Por que isso é um insight profundo ​

A maioria dos sistemas de retrieval baseados em grafo trata PPR como "propagação de relevance até convergir". O NEXUS trata PPR como "propagação de relevance por exatamente 3 hops".

A diferença parece de grau. É de natureza.

Convergência é um conceito de equilíbrio global. Ela responde: "Dado tempo infinito, onde o sistema descansa?" Isso é irrelevante para retrieval contextual, porque contexto é local.

Horizonte limitado é um conceito de vizinhança. Responde: "Dado um budget fixo de hops, quais nós são alcançáveis?" Isso é exatamente o que montagem de contexto precisa.

A armadilha conceptual é confundir "mais hops = mais contexto". Mais hops não expandem o contexto — eles diluem a query no ruído estrutural do grafo. Depois de 3 hops, qualquer nó adicional recebe tão pouco score que não contribui para o ranking. O limiar PPR_MIN_SCORE = 0.01 (main.rs:25) codifica isso matematicamente.

O projeto T4 seria inútil se retornasse "os nós mais conectados do grafo". Seria apenas um ranking por grau estrutural, não retrieval semântico. O horizonte de 3 hops é o mecanismo que preserva a direção da query através da propagação topológica.

Sem o horizonte limitado, o T4 seria um pagerank comum, dominado por hubs estruturais. Com o horizonte limitado, o T4 é um mecanismo de navegação geodésica que preserva locality semântica.
