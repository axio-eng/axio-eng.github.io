Formal Analysis of the NEXUS Hamiltonian Stress Function ​

Mathematical Properties, Implications, and Open Conjectures ​

Authors: Axi (AxiomaticDelirium), Bruno
Date: 2026-03
Status: WORKING PAPER — analytical results only; no experimental validation.
Companion document: ANALYSIS_REVIEW_V3_3.md

Preface ​

This document derives the mathematical properties of the canonical Hamiltonian stress function implemented in code/src/coordinator/src/mesh.rs. The goal is to establish which properties hold by proof, which fail by counter-example, and which remain open conjectures requiring empirical or formal verification.

No property is asserted without proof or explicit acknowledgment that it is a conjecture. Where results are negative (a claimed property fails), the failure is stated directly and its architectural implications are analyzed.

The Python formulation in scripts/nexus_topology.py is treated as a secondary object for comparison only. It is not the canonical implementation.

1. Formal Definition ​

1.1 Domain and Variables ​

Let $N \in \mathbb{N}$ be the embedding dimensionality ($N = 768$ in the current deployment using nomic-embed-text).

Define the following typed variables:

$$V_s, V_t \in \mathbb{R}^N \quad \text{(source and target node embeddings)}$$

$$T \in \mathbb{R}^N, \quad T_i > 0 ;\forall i \quad \text{(transformation diagonal)}$$

$$\delta \geq 0 \quad \text{(damping factor)}$$

$$w > 0 \quad \text{(base edge weight)}$$

$$\ell_s, \ell_t \in {0, 1, 2, 3, 4} \quad \text{(abstraction levels)}$$

$$M_s = 10^{\ell_s}, \quad M_t = 10^{\ell_t} \quad \text{(node masses)}$$

The Hadamard product $T \odot V_s \in \mathbb{R}^N$ is defined componentwise:

$$(T \odot V_s)_i = T_i \cdot (V_s)_i$$

1.2 The Canonical Hamiltonian ​

$$\boxed{H(V_s, V_t, T, \delta, w, \ell_s, \ell_t) = \max!\left(0,; \frac{|V_t - T \odot V_s|^2}{N} - \delta \right) \cdot \sqrt{M_s \cdot M_t} \cdot w}$$

This is a scalar-valued function $H : \mathbb{R}^N \times \mathbb{R}^N \times \mathbb{R}^N_{>0} \times \mathbb{R}{\geq 0} \times \mathbb{R}{>0} \times \mathbb{Z}^2 \to \mathbb{R}_{\geq 0}$.

1.3 Structural Decomposition ​

It is useful to decompose $H$ into three factors:

$$H = \underbrace{E(V_s, V_t, T, \delta)}{\text{effective energy}} \cdot \underbrace{\sqrt{M_s \cdot M_t}}{\text{mass coupling}} \cdot \underbrace{w}_{\text{base weight}}$$

where

$$E(V_s, V_t, T, \delta) = \max!\left(0,; \frac{|V_t - T \odot V_s|^2}{N} - \delta \right)$$

The mass coupling and base weight are strictly positive constants with respect to the embedding vectors. All analytical properties of $H$ as a function of $(V_s, V_t)$ are determined entirely by $E$.

1.4 The Comparison Object ​

The Python formulation (nexus_topology.py:85-95) is:

$$H_{\text{py}}(V_s, V_t, w) = (1 - \cos(V_s, V_t)) \cdot w \cdot |V_s - V_t|$$

where $\cos(V_s, V_t) = \frac{\langle V_s, V_t \rangle}{|V_s| \cdot |V_t|}$.

This object is analyzed in §7.

2. P1 — Differentiability ​

Claim: $H$ is differentiable with respect to $V_s$ and $V_t$ everywhere except on the set $\mathcal{S} = {(V_s, V_t) : |V_t - T \odot V_s|^2 / N = \delta}$. On $\mathcal{S}$, $H$ admits a well-defined subdifferential (Clarke generalized gradient).

Proof.

Define $g(V_s, V_t) = |V_t - T \odot V_s|^2 / N - \delta$. This is a quadratic function of $(V_s, V_t)$ and is therefore $C^\infty$ (smooth) everywhere.

$E = \max(0, g)$ is the composition of $\max(0, \cdot)$ with the smooth function $g$. The function $\max(0, \cdot) : \mathbb{R} \to \mathbb{R}$ is differentiable everywhere except at the origin, where it is non-differentiable but Lipschitz continuous. Therefore $E$ is differentiable everywhere except on $\mathcal{S}$.

The gradients on the two regions are:

$$\nabla_{V_s} H = \begin{cases} -\frac{2}{N} \cdot \sqrt{M_s M_t} \cdot w \cdot \text{diag}(T) \cdot (V_t - T \odot V_s) & \text{if } g > 0 \ 0 & \text{if } g < 0 \end{cases}$$

$$\nabla_{V_t} H = \begin{cases} \frac{2}{N} \cdot \sqrt{M_s M_t} \cdot w \cdot (V_t - T \odot V_s) & \text{if } g > 0 \ 0 & \text{if } g < 0 \end{cases}$$

On $\mathcal{S}$ (where $g = 0$), the Clarke subdifferential of $E$ with respect to a scalar input $x$ at $x = 0$ is the interval $[0, 1]$ (Rockafellar & Wets, 1998, §8). The subdifferential of $H$ on $\mathcal{S}$ is therefore the convex hull of the two one-sided limits of the gradient, which is a convex compact set.

Consequence for optimization: $H$ is amenable to subgradient methods (Shor, 1985). Any gradient-based optimization of graph topology (e.g., minimizing $H_G = \sum_e H(e)$ with respect to edge tensors $T$) can use subgradient descent with convergence guarantees under standard step-size conditions. $\blacksquare$

3. P2 — Convexity ​

Claim: $H$ is convex in $V_s$ given $(V_t, T, \delta, w, M_s, M_t)$ fixed, and convex in $V_t$ given $(V_s, T, \delta, w, M_s, M_t)$ fixed. $H$ is not jointly convex in $(V_s, V_t)$.

Proof (convexity in $V_s$).

Fix $V_t, T, \delta, w, M_s, M_t$. Define the affine map $\phi : \mathbb{R}^N \to \mathbb{R}^N$ by $\phi(V_s) = V_t - T \odot V_s$. This is affine in $V_s$.

The function $f(u) = |u|^2 / N$ is convex (it is a scaled squared norm, which is the canonical example of a strictly convex function). The composition $f \circ \phi$ is convex because $f$ is convex and $\phi$ is affine (Boyd & Vandenberghe, 2004, §3.2.2).

Subtracting the constant $\delta$ preserves convexity. Applying $\max(0, \cdot)$ preserves convexity because $\max(0, \cdot)$ is convex and nondecreasing (Boyd & Vandenberghe, 2004, §3.2.4). Multiplying by the positive constant $\sqrt{M_s M_t} \cdot w > 0$ preserves convexity.

Therefore $H$ is convex in $V_s$. By symmetry of the argument (with the roles of $V_s$ and $V_t$ exchanged and $T$ absorbed into the definition), $H$ is convex in $V_t$.

Proof (failure of joint convexity).

Joint convexity would require the Hessian of $H$ with respect to $(V_s, V_t) \in \mathbb{R}^{2N}$ to be positive semidefinite. In the region $g > 0$, the Hessian is:

$$\nabla^2 H = \frac{2\sqrt{M_s M_t} \cdot w}{N} \begin{pmatrix} \text{diag}(T)^2 & -\text{diag}(T) \ -\text{diag}(T) & I \end{pmatrix}$$

For this $2N \times 2N$ block matrix to be PSD, the Schur complement condition requires $I - \text{diag}(T) \cdot \text{diag}(T)^{-2} \cdot \text{diag}(T) = I - I = 0$ to be PSD, which holds — but only when $T = \mathbf{1}$ (the identity). For general $T$ with $T_i \neq 1$ for some $i$, the off-diagonal coupling $-\text{diag}(T)$ induces indefiniteness.

Counter-example (N=1): Let $N = 1$, $T = 2$, $\delta = 0$, $w = 1$, $M_s = M_t = 1$. Then $H(V_s, V_t) = (V_t - 2V_s)^2$. The Hessian is $\begin{pmatrix} 4 & -4 \ -4 & 1 \end{pmatrix}$, with eigenvalues $\frac{5 \pm \sqrt{25 - 4(4 \cdot 1 - 16)}}{2} = \frac{5 \pm \sqrt{73}}{2}$. Since $\sqrt{73} > 5$, one eigenvalue is negative. The Hessian is indefinite. $\blacksquare$

Architectural consequence: $H$ is separately convex but not jointly convex. Alternating optimization (fix $V_t$, optimize $V_s$; then fix $V_s$, optimize $V_t$) is guaranteed to converge to a local minimum. Joint optimization has no such guarantee. This is relevant if future work attempts to learn optimal node embeddings by minimizing $H_G$.

4. P3 — Asymmetry and the Triangle Inequality ​

This is the most consequential result for the T-RAG architecture.

Claim A (Asymmetry): $H$ is not symmetric. In general, $H(V_s, V_t, T, \cdot) \neq H(V_t, V_s, T, \cdot)$.

Proof. $H(V_s, V_t) \propto |V_t - T \odot V_s|^2$ while $H(V_t, V_s) \propto |V_s - T \odot V_t|^2$.

These are equal if and only if $|V_t - T \odot V_s| = |V_s - T \odot V_t|$, which holds when $T = \mathbf{1}$ but fails in general. Counter-example: $N=1$, $V_s = 1$, $V_t = 0$, $T = 2$. Then $|V_t - T \odot V_s|^2 = (0 - 2)^2 = 4$ while $|V_s - T \odot V_t|^2 = (1 - 0)^2 = 1$. $\blacksquare$

Claim B (Triangle inequality fails): $H$ does not satisfy the triangle inequality. That is, there exist nodes $s, t, u$ and tensors $\mathcal{T}{su}, \mathcal{T}, \mathcal{T}_{tu}$ such that $H(s, u) > H(s, t) + H(t, u)$.

Counter-example.

Take $N = 1$, $T = \mathbf{1}$, $\delta = 0$, $w = 1$. Then $H(s,t) = (V_t - V_s)^2 \cdot \sqrt{M_s M_t}$.

Let $\ell_s = 0$, $\ell_t = 4$, $\ell_u = 0$ (so $M_s = 1$, $M_t = 10^4$, $M_u = 1$). Let $V_s = 0$, $V_t = 1$, $V_u = 2$.

$$H(s, u) = (2 - 0)^2 \cdot \sqrt{1 \cdot 1} = 4$$

$$H(s, t) = (1 - 0)^2 \cdot \sqrt{1 \cdot 10^4} = 1 \cdot 100 = 100$$

$$H(t, u) = (2 - 1)^2 \cdot \sqrt{10^4 \cdot 1} = 1 \cdot 100 = 100$$

$$H(s,t) + H(t,u) = 200 \gg 4 = H(s,u)$$

This direction fails in the expected way (the direct path is cheaper). Now reverse the embedding values to find the other direction:

Let $V_s = 0$, $V_t = 0.01$, $V_u = 0.02$.

$$H(s,u) = (0.02)^2 \cdot 1 = 0.0004$$ $$H(s,t) = (0.01)^2 \cdot 100 = 0.01$$ $$H(s,t) + H(t,u) = 0.02 \gg 0.0004$$

The mass coupling at L4 amplifies every edge incident to $t$ by a factor of $10^4$. The direct $s \to u$ path (both L0, mass coupling = 1) is always cheaper than any path through an intermediate L4 node. The triangle inequality is violated whenever an intermediate high-mass node exists on any path between two low-mass nodes.

This is not a numerical accident. It is a structural consequence of the mass coupling. The exponential mass hierarchy ($M = 10^\ell$) ensures that high-level nodes act as energy amplifiers. Any path through them is energetically expensive relative to a direct low-level path, even if the direct path spans a larger embedding distance. $\blacksquare$

4.1 Implications for PPR ​

The triangle inequality failing does not invalidate PPR. Andersen et al. (2006) prove convergence of approximate PPR for any graph with non-negative edge weights — no metric structure is required.

However, the asymmetry (Claim A) has a direct consequence: the random walk underlying PPR is on a directed graph. PPR on directed graphs has weaker convergence guarantees than on undirected graphs. Specifically:

  1. The stationary distribution may not be unique if the graph has multiple strongly connected components with different weight structures.
  2. The restart probability $\alpha$ required for convergence depends on the spectral gap of the directed transition matrix, which is harder to bound than in the undirected case.
  3. If the directed graph contains sink nodes (nodes with no outgoing edges of positive weight), the random walk absorbs probability mass at those sinks, distorting the PPR distribution.

Architectural implication: T-RAG as described in §3.5 of ANALYSIS_REVIEW_V3_3.md implicitly assumes that PPR convergence on the directed, mass-weighted NEXUS graph inherits the properties of standard PPR. This assumption is unverified. The convergence analysis of PPR on directed graphs with the NEXUS mass weighting is an open problem (see §9.1).

4.2 The Asymmetry as a Feature ​

The asymmetry is architecturally interpretable. $H(s \to t) \neq H(t \to s)$ encodes directional semantic displacement: the cost of mapping $V_s$ toward $V_t$ via tensor $T$ is not the same as the cost of the reverse mapping. This is semantically meaningful for typed relations: the stress of "A implements B" is not the same as the stress of "B implements A". Forcing symmetry would discard this information.

The design question is not "should we make $H$ symmetric?" but "does PPR on this directed graph converge to a useful distribution?" That is an empirical question.

5. P4 — Rotational Invariance ​

Claim: $H$ is invariant under simultaneous orthogonal rotation of $V_s$ and $V_t$ if and only if $T = \mathbf{1}$ (the all-ones vector, i.e., the identity Hadamard operator).

Proof.

Let $R \in \mathbb{R}^{N \times N}$ be an orthogonal matrix ($R^T R = I$). Consider the effect of replacing $(V_s, V_t) \mapsto (RV_s, RV_t)$:

$$|RV_t - T \odot (RV_s)|^2 = |RV_t - (T \odot I) R V_s|^2$$

When $T = \mathbf{1}$, $T \odot I = I$ (componentwise multiplication of a matrix by a vector of ones is the identity operation on diagonal entries, which here means the operation is just $R$). Therefore:

$$|(RV_t) - R(V_s)|^2 = |R(V_t - V_s)|^2 = |V_t - V_s|^2$$

using the isometric property of orthogonal matrices ($|Rx| = |x|$). So $H$ is rotationally invariant when $T = \mathbf{1}$.

When $T \neq \mathbf{1}$, the Hadamard product $T \odot (RV_s)$ applies $T$ componentwise to the rotated vector $RV_s$. This is not equal to $R(T \odot V_s)$ in general, because $T$ is diagonal in the original basis, not the rotated basis. Specifically:

$$T \odot (RV_s) \neq R(T \odot V_s) \quad \text{when } T \neq c\mathbf{1} \text{ for any scalar } c$$

Counter-example ($N=2$): Let $T = (2, 1)^T$, $V_s = (1, 0)^T$, $V_t = (0, 1)^T$, and $R$ be the $90°$ rotation matrix $\begin{pmatrix} 0 & -1 \ 1 & 0 \end{pmatrix}$.

Original: $T \odot V_s = (2, 0)^T$. $|V_t - T \odot V_s|^2 = |(0-2, 1-0)|^2 = 4 + 1 = 5$.

Rotated: $RV_s = (0, 1)^T$, $RV_t = (-1, 0)^T$. $T \odot (RV_s) = (2 \cdot 0, 1 \cdot 1)^T = (0, 1)^T$. $|RV_t - T \odot (RV_s)|^2 = |(-1-0, 0-1)|^2 = 1 + 1 = 2$.

$5 \neq 2$. $H$ is not rotationally invariant. $\blacksquare$

5.1 Architectural Consequence: Model-Specificity of Tensor Edges ​

The tensor $T$ encodes semantic displacement in the coordinate system of the embedding model. When the embedding model changes (e.g., replacing nomic-embed-text with a different model), the embedding space undergoes a transformation that is generally not an orthogonal rotation — it is an arbitrary nonlinear transformation.

Consequence: All stored tensor edges $T$ become invalid when the embedding model is replaced. The stress values computed with old tensors and new embeddings are semantically meaningless.

This is not a bug — it is a structural property of the architecture. It implies that tensor edges are versioned artifacts, coupled to the embedding model that generated the embeddings they operate on. Any model migration requires recomputation of all T4 tensor edges from the new T3 embedding space.

This should be reflected in the schema: the links table should carry a embedding_model column that records which model was active when the tensor was computed.

6. P5 — Asymptotic Behavior and the Critical Damping Threshold ​

6.1 The $\delta^*$ Critical Point ​

For a fixed edge $(s, t, T)$, define the raw normalized energy:

$$E_0(s, t, T) = \frac{|V_t - T \odot V_s|^2}{N}$$

The effective energy $E = \max(0, E_0 - \delta)$ is zero if and only if $\delta \geq E_0$. Define the critical damping threshold for this edge:

$$\delta^*(s, t, T) = E_0(s, t, T) = \frac{|V_t - T \odot V_s|^2}{N}$$

For $\delta \geq \delta^$: $H = 0$. The edge carries zero stress.
For $\delta < \delta^$: $H = (\delta^* - \delta) \cdot \sqrt{M_s M_t} \cdot w > 0$.

$H$ is linear in $\delta$ (for $\delta < \delta^*$) with slope $-\sqrt{M_s M_t} \cdot w$.

6.2 Global Damping Collapse ​

If a single global $\delta$ is applied to all edges (as implied by the current implementation where damping_factor is a per-tensor field but may be initialized uniformly), then for any $\delta \geq \max_{(s,t,T) \in E} \delta^*(s, t, T)$, the global graph energy is:

$$H_G = \sum_{e \in E} H(e) = 0$$

The graph is energetically silent — no tension, no ruptures, no gradient signal. This is a degenerate configuration.

Practical consequence: The damping factor $\delta$ must be calibrated to be strictly below $\delta^*$ for the majority of edges. If $\delta$ is set too high (e.g., as a noise floor estimate based on embedding variance), the system silently loses all discriminative power. There is no error — $H = 0$ is a valid output — but the semantic monitoring function degrades to a no-op.

6.3 Bounds on $\delta^*$ ​

For nomic-embed-text embeddings, the vectors are approximately unit-normalized: $|V| \approx 1$. When $T = \mathbf{1}$:

$$E_0 = \frac{|V_t - V_s|^2}{N} = \frac{2(1 - \cos(V_s, V_t))}{N}$$

The range of $E_0$ is $[0, 4/N]$ (since $|V_t - V_s|^2 \leq (|V_t| + |V_s|)^2 \approx 4$ for unit vectors). For $N = 768$: $E_0 \in [0, 0.0052]$.

This is a very small range. A uniform $\delta = 0.001$ would suppress approximately half of all edges (those with cosine similarity $> 0.62$). A $\delta = 0.005$ would suppress nearly all edges. The damping factor must be set in the range $[0, 4/N]$ — for $N = 768$, this means $\delta \in [0, 0.0052]$ — to be effective.

This range is empirically small and unintuitive. It should be exposed as a calibrated hyperparameter, not hardcoded.

7. Comparison: $H_{\text{Rust}}$ vs $H_{\text{py}}$ ​

7.1 Analytical Relationship for Unit Vectors ​

For approximately unit-normalized embeddings ($|V_s| \approx |V_t| \approx 1$), with $T = \mathbf{1}$ and $\delta = 0$, the Rust Hamiltonian simplifies to:

$$H_{\text{Rust}} = \frac{|V_t - V_s|^2}{N} \cdot \sqrt{M_s M_t} \cdot w = \frac{2(1 - \cos)}{N} \cdot \sqrt{M_s M_t} \cdot w$$

using the identity $|V_t - V_s|^2 = 2 - 2\cos$ for unit vectors.

The Python formulation, with $M_s = M_t = 1$ (no mass) and unit vectors:

$$H_{\text{py}} = (1 - \cos) \cdot w \cdot |V_s - V_t| = (1 - \cos) \cdot w \cdot \sqrt{2(1-\cos)} = w\sqrt{2} \cdot (1-\cos)^{3/2}$$

Setting $x = 1 - \cos \in [0, 2]$:

$$H_{\text{Rust}} \propto \frac{2x}{N} \qquad H_{\text{py}} \propto \sqrt{2} \cdot x^{3/2}$$

7.2 Properties of the Difference ​

Both functions are monotonically increasing in $x = 1 - \cos$ for $x \geq 0$. Therefore, they produce the same ranking of edges by stress — for unit vectors, without tensor transformation and without damping.

They differ in:

Property$H_{\text{Rust}}$$H_{\text{py}}$
Growth rate in $x$Linear: $O(x)$Super-linear: $O(x^{3/2})$
Sensitivity near $x=0$Constant slope $2/N$Zero slope at $x=0$ (derivative = $\frac{3\sqrt{2}}{2}\sqrt{x} \to 0$)
Sensitivity near $x=2$Slope $2/N$Slope $3\sqrt{2} \cdot \sqrt{2}/2 = 3$
Mass couplingYesNo
Tensor transformationYesNo
DampingYesNo

The key structural difference: $H_{\text{py}}$ has zero first derivative at $x=0$ — it is locally flat near perfect similarity. $H_{\text{Rust}}$ has a constant positive slope at $x=0$. This means $H_{\text{Rust}}$ is more sensitive to small deviations from perfect alignment, while $H_{\text{py}}$ is more sensitive to large deviations.

For the purpose of contradiction detection (flagging semantically inconsistent but topologically connected nodes), $H_{\text{Rust}}$ is preferable: it amplifies small perturbations that $H_{\text{py}}$ would suppress near zero.

7.3 Verdict ​

The two formulations produce the same edge ranking for unit vectors in the degenerate case ($T = \mathbf{1}$, $\delta = 0$, no mass). In all other cases — which include all realistic operating conditions — they are mathematically incomparable. The Rust formulation is richer (tensor transformation, mass coupling, damping) and is the canonical implementation. The Python formulation is a prototype exploration that should not be used to compute values that will be compared to, or combined with, Rust-computed stress values.

8.2 Directed Graph and PPR ​

The asymmetry of $H$ means the NEXUS knowledge graph is inherently directed. The edge $s \to t$ has stress $H(s,t)$ and the edge $t \to s$ has stress $H(t,s)$, and these are different values computed by the same tensor (which encodes a directed relation).

For PPR, the directed graph has the following implication: the transition matrix $P_{ij} = \exp(-H(i,j)/\tau) / Z_i$ (where $Z_i$ normalizes row $i$) is the transition matrix of a directed random walk. Personalized PageRank on directed graphs converges to a unique stationary distribution if and only if the graph is strongly connected — every node is reachable from every other node via directed paths (Gleich, 2015).

The NEXUS graph is unlikely to be strongly connected in practice: L4 nodes have authority over L0 nodes (edges flow downward in level), and the reverse direction may have no edges. If the graph has multiple strongly connected components (SCCs), the PPR distribution will be absorbed by the sink SCCs.

Practical mitigation: Add low-weight "gravity" edges from every node back to its abstraction source (the blob it was derived from), ensuring weak strong-connectivity. This is an architectural decision, not a mathematical one.

8.3 The $\delta$ Calibration Problem ​

The critical damping threshold $\delta^* \in [0, 4/N] = [0, 0.0052]$ for unit-normalized embeddings is a tight constraint. The damping factor must be set empirically by examining the distribution of $E_0$ values across the actual corpus. This distribution is unknown until T3 is fully populated.

Recommendation: Treat $\delta$ as a corpus-dependent hyperparameter. After T3 population, compute the empirical distribution of $E_0 = |V_t - T \odot V_s|^2 / N$ across all potential edges. Set $\delta$ at the $p$-th percentile of this distribution (e.g., $p = 10$), suppressing the 10% of edges with the smallest energy while preserving 90% of the signal.

9. Open Conjectures ​

The following conjectures are stated as formal hypotheses. They require either mathematical proof or empirical validation. None is asserted as true.

9.1 PPR Convergence on the NEXUS Directed Graph ​

Conjecture: Let $G_\tau = (V, E, W_\tau)$ be the NEXUS graph with edge weights $W_\tau(s,t) = \exp(-H(s,t)/\tau)$ for temperature $\tau > 0$. For any restart distribution $\pi_0$ concentrated on a seed set $S \subset V$, Personalized PageRank on $G_\tau$ with restart probability $\alpha \in (0,1)$ converges to a unique stationary distribution $\pi^*$ as the number of iterations $k \to \infty$.

Status: Unproven. Convergence is guaranteed if $G_\tau$ is strongly connected. Whether the NEXUS graph is strongly connected depends on the edge structure, which is empirically determined. The conjecture may be false for sparse graphs or graphs with isolated high-level nodes.

Path to resolution: After T4 population, compute the SCC decomposition of the graph. If strongly connected: convergence is guaranteed by standard Markov chain theory. If not: characterize the sink SCCs and either (a) prove that PPR still produces useful rankings on the reachable subgraph, or (b) add structural edges to enforce strong connectivity.

9.2 $H_G$ as a Consistency Indicator ​

Conjecture: Let $H_G = \sum_{e \in E} H(e)$ be the total graph energy. Then $H_G$ is a monotone indicator of semantic inconsistency: graphs containing more contradictory node pairs (as annotated by human raters) have higher $H_G$.

Status: Unproven. This is Hypothesis H1 from PEER_REVIEW_V3_3.md. It requires a human-annotated dataset of contradictory node pairs and a controlled experiment comparing $H_G$ before and after introducing known contradictions.

Formal falsification criterion: If there exists a graph $G'$ obtained from $G$ by adding a known contradiction $(s, t)$ such that $H_{G'} < H_G$, the conjecture is false. This could happen if the added contradiction introduces high-weight edges that, through the mass coupling, reduce the normalized energy of neighboring edges.

9.3 T-RAG Retrieval Quality ​

Conjecture: For a retrieval task with ground-truth relevance labels, T-RAG (PPR with weights $W_\tau(e) = \exp(-H(e)/\tau)$) achieves higher NDCG@k than flat k-NN retrieval from T3.

Status: Unproven. This is Hypothesis H2 from PEER_REVIEW_V3_3.md. It requires T4 to be populated and T-RAG to be implemented.

Note on the conjecture's scope: This conjecture is about retrieval quality, not about any property of $H$ itself. Even if $H$ has all the analytical properties described in this document, T-RAG could still underperform k-NN if the graph structure adds noise rather than signal. The conjecture is empirical, not analytical.

9.4 Optimal Mass Exponent ​

Conjecture: The mass function $M(\ell) = 10^\ell$ is optimal in the sense that it maximizes the separation between stress distributions at different level pairs $(\ell_s, \ell_t)$ while minimizing variance within each level pair.

Status: Unproven. The base $10$ is an engineering choice. The optimal base — defined with respect to the empirical distribution of embedding distances at each level — is unknown. It is possible that a smaller base (e.g., $e$ or $2$) produces better-calibrated stress values, or that the optimal base varies by level pair.

9.5 Tensor Edge Learnability ​

Conjecture: Given a fixed set of node embeddings ${V_i}$ and a set of human-annotated edge labels (e.g., "contradicts", "implements", "extends"), there exists a learning procedure that recovers tensor diagonals $T_{ij}$ such that $H(V_i, V_j, T_{ij})$ is discriminative for the labeled relation types.

Status: Unproven. This is the core learning problem for T4. It is related to the problem of learning relation-specific transformations in knowledge graph embedding (cf. TransR, Lin et al., 2015), but in the NEXUS setting the objective function is the Hamiltonian stress rather than a margin-based ranking loss.
