Data: 2026-06-09 Contexto: Sessão pós-L3, refletindo sobre stateless × stateful para operadores cognitivos

2. O Insight Central: Stateless × Stateful ​

O problema: NEXUS hoje é uma mistura de bancos (PG, Qdrant), representações (SQL, vetores, grafos), e níveis de abstração (L0-L3). Um LLM agente chega e recebe um nexus_guide() estático — um mapa impresso. Ele não pode "andar pelo prédio".

A descoberta: O operador de zoom precisa ser stateful porque:

  • Um agente quer explorar progressivamente: começar com uma visão compacta (L3), aprofundar onde interessa (L2, L1, L0)
  • Cada passo de zoom é uma transição de estado: o foco atual, a profundidade, o caminho percorrido
  • A compactação do conhecimento (abstração→detalhe) é inerentemente sequencial e dependente de contexto

Implicação: Stateful zoom transcende "só mais uma tool MCP". É um protocolo novo — o agente não apenas consulta, ele navega.

3. Protocolo Stateful de Zoom (Rascunho) ​

Estado por Sessão ​

python
{
  "session_id": "z_`&lt;uuid&gt;`",
  "query_original": "como funciona o enriquecimento L2",
  "foco_atual": "abstraction_id_l3",
  "arvore": {
    "raiz": "l3_id",
    "expandidos": ["l2_id_1", "l2_id_2"],
    "arestas": [("l3_id", "l2_id_1", "derived_from")]
  },
  "profundidade": 1,
  "contexto_acumulado": "## ... texto formatado ..."
}

Operadores ​

OperadorFunçãoMuda Estado?
nexus_zoom_init(query, depth_inicial=0)Cria sessão, encontra L3 match, retorna contextoSim (cria)
nexus_zoom_deepen(session_id, foco=None)Walk T4 1-hop do foco, expande árvoreSim (expande)
nexus_zoom_narrow(session_id, node_id)Move foco para nó específicoSim (redefine)
nexus_zoom_context(session_id, as_prompt=True)Serializa árvore como bloco de contextoNão
nexus_zoom_reset(session_id)Destrói sessãoSim (destrói)
nexus_zoom_list()Lista sessões ativasNão

O Que O Estado Permite ​

  1. Exploração progressiva: O agente pode deepen/narrow sem perder o histórico
  2. Contexto cumulativo: A cada passo, o contexto formatado cresce — o agente sempre vê o caminho completo
  3. Poda inteligente: Narrow em um nó específico descarta galhos paralelos que não interessam
  4. Reentrância: Se o agente perder contexto (janela cheia), pode recarregar via nexus_zoom_context

4. A Pergunta Fundamental: O Que Um LLM Precisa Saber Sobre o "Prédio"? ​

Quando um agente LLM chega no NEXUS, ele recebe:

  1. AGENTS.md — instruções estáticas (não fazer grep, chamar nexus_guide, etc.)
  2. nexus_guide() — o protocolo operacional
  3. nexus_map() — a planta baixa estrutural

O que falta: uma representação do conhecimento do domínio que seja:

  • Compacta (abraça o prédio inteiro em poucos tokens)
  • Navegável (cada conceito é um link que o agente pode "clicar")
  • Multiescala (do conceito mais geral ao detalhe mais específico)

Exemplo: O Welcome Prompt Ideal ​

## NEXUS — Knowledge Graph

Domínios: enrichment, query, storage, graph, ingestion
Níveis: L3 (conceitos) → L2 (significado) → L1 (estrutura) → L0 (código)

### enrichment (L3)
O pipeline de enriquecimento gera L1 (AST) e L2 (chunks semânticos)
a partir de L0 (código fonte). Usa Ollama para gerar propósito + resumo,
então embeda em Qdrant e popula agent_links (T4).

[zoom → enrichment/l2/detail]
[zoom → enrichment/l1/structure]
[zoom → enrichment/l0/source]

### query (L3)
T-RAG opera em 3 tiers: T2 (filtro SQL), T3 (k-NN Qdrant), T4 (PPR walk).
Rotear query simples vs complexa baseado em...

[zoom → query/t2]
[zoom → query/t3]  
[zoom → query/t4]

Cada [zoom → ...] é um link que o agente pode seguir com nexus_zoom_deepen(). O welcome prompt é a raiz da árvore de conhecimento.

5. Compactação por Abstração ​

A chave para o "swap biológico" (substituir componentes sem quebrar o ecossistema) é que o protocolo de zoom opera em abstrações, não em implementações:

Protocolo zoom (stateful)
       ↕
Abstrações L3 (conceitos independentes de storage)
       ↕
Implementações: PG | Qdrant | Ollama | agent_links

Se amanhã trocar Qdrant por Milvus, o protocolo de zoom não muda — só o backend de busca semântica. As L3 abstrações continuam sendo os nós navegáveis. O "swap" acontece abaixo da linha de abstração.
