ADR: NEXUS Ingestion Frontmatter Schema ​

Data: 24 de Março de 2026
Status: Proposto
Autores: Arquiteto NEXUS & Prometheus

1. Resumo ​

Este documento define o schema de frontmatter YAML para arquivos Markdown processados pelo NEXUS. O frontmatter permite que cada arquivo declare explicitamente como deve ser tratado pelo pipeline de ingestão, permitindo controle granular sobre indexação, priorização e categorização.

O schema foi projetado para ser backwards-compatible: arquivos sem frontmatter são tratados como ingestible: true por padrão.

2. Formato do Frontmatter ​

O frontmatter deve usar blocos YAML delimitados por ---:

yaml
---
nexus:
  ingestible: true
  priority: medium
  category: architecture
  tags: [api, rest, backend]
  horizon: stable
---

# Conteúdo do arquivo...

3. Campos Definidos ​

3.1 nexus.ingestible ​

Tipo: Boolean
Default: true (se ausente)
Valores válidos: true, false

Descrição:
Controla se o arquivo deve ser indexado no banco vetorial (Qdrant) e processado para gerar abstrações L1.

Por que é necessário:
Previne inflação circular do banco vetorial. Documentação sobre o NEXUS (ADRs,高层 docs) não deve ser ingerida porque:

  • Criaria loop de auto-referência semântica
  • Documentação é saída, não entrada para o sistema
  • Arquivos de workspace/resultados também não devem ser ingeridos

Exemplos:

yaml
# Excluir explicitamente (documentação, resultados)
nexus:
  ingestible: false

# Incluir (padrão para código e scripts)
nexus:
  ingestible: true

3.2 nexus.priority ​

Tipo: String (enum)
Default: medium
Valores válidos: low, medium, high, critical

Descrição:
Define a prioridade de processamento do arquivo no pipeline de ingestão.

Por que é necessário:
Permite roteamento diferenciado:

  • critical: arquivos de contrato, interfaces, APIs
  • high: lógica de negócio principal, código core
  • medium: utilities, helpers, configurações
  • low: arquivos auxiliares, migrations, seeds

Isso permite que o sistema gaste mais recursos (janela de tokens maior, modelo maior) em arquivos mais importantes.

Mapeamento de Complexidade (de ADR 006):

PriorityTokens LidosStrideModelo Ideal
critical~4.00057B - 8B params
high~2.500151B - 3B params
medium~2.500151B - 3B params
low~50030100M - 1B params

Exemplos:

yaml
# Contrato de API - prioridade máxima
nexus:
  ingestible: true
  priority: critical
  category: api
  horizon: stable

# Migration - prioridade baixa
nexus:
  ingestible: true
  priority: low
  category: database
  horizon: deprecated

3.3 nexus.category ​

Tipo: String
Default: undefined
Valores válidos: qualquer string (recomendado: api, architecture, database, frontend, backend, devops, docs, scripts, tests, config)

Descrição:
Categoria semântica do arquivo para organização e filtragem.

Por que é necessário:
Permite:

  • Consulta filtrada por categoria (ex: "me mostre toda documentação de API")
  • Agregação de métricas por categoria
  • Roteamento para LoRAs especializados por domínio
  • Identificação de gaps de documentação

Exemplos:

yaml
# Documentação de arquitetura
nexus:
  ingestible: true
  category: architecture
  tags: [adr, decisions]

# Script de migração
nexus:
  ingestible: true
  category: database
  tags: [migration, postgres]

# Componente React
nexus:
  ingestible: true
  category: frontend
  tags: [react, component, ui]

3.4 nexus.tags ​

Tipo: Array of Strings
Default: [] (array vazio)
Valores válidos: qualquer array de strings

Descrição:
Tags livres para organização granular e busca semântica.

Por que é necessário:
Categories são rígidas (um arquivo tem uma categoria), mas tags permitem múltiplas dimensões:

  • Tecnologias específicas: [rust, tokio, async]
  • Padrões: [singleton, factory, observer]
  • Domínios: [auth, payments, users]
  • Status: [deprecated, experimental, stable]

Exemplos:

yaml
# API REST com autenticação JWT
nexus:
  ingestible: true
  category: api
  tags: [rest, jwt, auth, openapi]

# Função utilitária de data
nexus:
  ingestible: true
  category: utils
  tags: [date, time, formatting, i18n]

# Depreciado mas ainda referenciado
nexus:
  ingestible: true
  category: config
  tags: [deprecated, legacy, v1]

3.5 nexus.horizon ​

Tipo: String (enum)
Default: stable
Valores válidos: stable, beta, alpha, deprecated, experimental

Descrição:
Indica o horizonte temporal/estabilidade do conteúdo.

Por que é necessário:
Permite gestão de ciclo de vida:

  • stable: contratos, APIs publicadas, documentação final
  • beta: em teste, pode mudar
  • alpha: fortemente em mudança
  • deprecated: mantido por compatibilidade, slated para remoção
  • experimental: provas de conceito, pode ser removido sem aviso

Isso permite que consultas priorizem conteúdo relevante para o contexto operacional atual e evita "drift" semântico com código obsoleto.

Exemplos:

yaml
# API pública estável
nexus:
  ingestible: true
  category: api
  horizon: stable

# Feature em desenvolvimento ativo
nexus:
  ingestible: true
  category: feature
  horizon: alpha

# Wrapper legado
nexus:
  ingestible: true
  category: adapter
  horizon: deprecated

4. Comportamento Default e Backwards Compatibility ​

Regra Fundamental ​

Arquivos sem frontmatter são tratados como ingestible: true.

Isso garante que:

  • Arquivos existentes continuem sendo processados normalmente
  • O sistema não quebre se frontmatter for acidentalmente removido
  • A migração seja gradual e opcional

Fast-Scan ​

Para performance, o parsing de frontmatter deve usar fast-scan:

  1. Ler apenas os primeiros 1KB do arquivo
  2. Verificar se começa com ---
  3. Se não começar com ---, retornar defaults (ingestible: true)
  4. Só fazer parsing completo se necessário

6. Implementação ​

6.1 Estrutura de Dados (Rust) ​

rust
#[derive(Debug, Deserialize)]
pub struct NexusMetadata {
    #[serde(default = "default_ingestible")]
    pub ingestible: bool,
    
    #[serde(default = "default_priority")]
    pub priority: String,
    
    #[serde(default)]
    pub category: Option`&lt;String&gt;`,
    
    #[serde(default)]
    pub tags: Vec`&lt;String&gt;`,
    
    #[serde(default = "default_horizon")]
    pub horizon: String,
}

fn default_ingestible() -&gt; bool { true }
fn default_priority() -&gt; String { "medium".to_string() }
fn default_horizon() -&gt; String { "stable".to_string() }

6.2 Armazenamento (PostgreSQL JSONB) ​

Os campos são armazenados no JSONB metadata da tabela abstractions:

json
{
  "nexus": {
    "ingestible": true,
    "priority": "high",
    "category": "api",
    "tags": ["rest", "auth"],
    "horizon": "stable"
  },
  "source_chunks": 3,
  "window_size": 2500
}

6.3 Gating no Coordinator ​

O is_excluded deve verificar nexus.ingestible antes de processar:

rust
pub fn is_excluded(path: &Path, content: &str) -&gt; bool {
    // 1. Check path-based exclusions (existing logic)
    if is_path_excluded(path) {
        return true;
    }
    
    // 2. Check frontmatter ingestible flag
    if let Some(metadata) = extract_nexus_metadata(content) {
        if !metadata.ingestible {
            return true;
        }
    }
    
    false
}

7. Decisões de Design ​

7.1 Campos Fixos vs. Livres ​

Optamos por campos fixos com semantics definidas ao invés de JSONB livre porque:

  • Permite validação em tempo de parse
  • Facilita querying estruturado no PostgreSQL
  • Proporciona consistency entre arquivos
  • Permite otimização de storage (indexação)

7.2 Nomenclatura nexus.* ​

Prefixo nexus. evita colisão com outros frontmatter populares (Hugo, Jekyll, VS Code) e deixa claro que esses campos são para o sistema NEXUS.

7.3 Enums como Strings ​

Usamos strings para priority e horizon ao invés de enums estritos para:

  • Extensibilidade (novos valores podem ser adicionados sem breaking change)
  • Simplicidade de parsing (serde_yaml handle strings natively)
  • Flexibilidade operacional (scripts podem gerar valores customizados se necessário)

8. Migração de Arquivos Existentes ​

Arquivos existentes sem frontmatter devem ser gradualmente migrados:

DiretórioRecomendação
scripts/ingestible: true, priority conforme análise
services/ingestible: true, priority conforme análise
docs/ingestible: false (são saídas)
docs/archive/ingestible: false
workspace/ingestible: false (já excluído)

Ver: scripts/migrate_docs_frontmatter.py
