Identidade GCP do NEXUS — nexus-share ​

Propósito: Compartilhar recursos do workspace NEXUS com a conta pessoal do operador (bmbanho@gmail.com) — Cloud Storage, Vertex AI, BigQuery, e serviços Google.

A chave está FORA do repositório. Nenhum agente deve copiá-la para dentro do workspace.

1. A Identidade ​

AtributoValor
Service Accountnexus-share@stone-history-499211-k9.iam.gserviceaccount.com
Projetostone-history-499211-k9
Chave JSON~/.config/gcloud/nexus-share-key.json (modo 600)
Profile Script~/.config/gcloud/nexus-profile.sh

Permissões ​

RoleEscopo
roles/storage.adminGCS — ler, escrever, criar buckets, definir IAM
roles/aiplatform.userVertex AI — modelos, notebooks, pipelines, datasets
roles/bigquery.userBigQuery — executar consultas

O operador (bmbanho@gmail.com) tem permissão de impersonation (roles/iam.serviceAccountTokenCreator + roles/iam.serviceAccountUser) sobre a SA, permitindo uso sem chave quando conveniente.

2. Quick Start ​

bash
# 1. Carregar o perfil
source ~/.config/gcloud/nexus-profile.sh

# 2. Ativar a identidade
nexus::activate

# 3. Verificar
nexus::whoami

# 4. Depois de usar, voltar pra conta pessoal
nexus::restore

3. Comandos do Perfil ​

Todos os comandos ficam disponíveis após source ~/.config/gcloud/nexus-profile.sh.

4. Uso para Agentes ​

Agentes no workspace NEXUS precisam acessar recursos GCP durante operação. O fluxo correto:

4.1 Descoberta ​

O profile script está em ~/.config/gcloud/nexus-profile.sh — fora do repositório. Agentes devem:

  1. Verificar se o arquivo existe: test -f ~/.config/gcloud/nexus-profile.sh
  2. Se existir, fazer source para carregar as funções
  3. Usar nexus::adc::use-sa para configurar ADC para SDKs

4.2 Para comandos gcloud diretos ​

bash
source ~/.config/gcloud/nexus-profile.sh
nexus::activate
# ... comandos gcloud ...
nexus::restore

4.3 Para SDKs (Python, Node.js, etc.) ​

bash
source ~/.config/gcloud/nexus-profile.sh
nexus::adc::use-sa
# SDKs agora autenticam como nexus-share automaticamente
# Ex: python3 -c "from google.cloud import storage; client = storage.Client()"

4.4 Para operações que precisam da conta pessoal ​

A SA pode criar e compartilhar recursos com bmbanho@gmail.com. Padrão:

bash
# 1. Como SA: criar bucket
nexus::gcs::create-bucket "nexus-share-$USER-$(date +%s)"

# 2. Como SA: compartilhar com a conta pessoal
nexus::gcs::share "nexus-share-$USER-..." "bmbanho@gmail.com" "roles/storage.objectAdmin"

# 3. Agora a conta pessoal tem acesso direto pelo Console/CLI

4.5 Impersonation (sem chave) ​

Se o agente está autenticado como bmbanho@gmail.com:

bash
gcloud storage ls --impersonate-service-account=nexus-share@stone-history-499211-k9.iam.gserviceaccount.com

Válido para qualquer comando gcloud. Útil quando a chave não está disponível.

A SA age como ponte: cria recursos no projeto e concede acesso à conta pessoal.

6. Gemini API Keys — Round-Robin ​

O Problema ​

Gemini API keys só funcionam em projetos criados por contas Google comuns (@gmail.com). Contas Google Workspace (@axio.eng.br) são bloqueadas pela API generativelanguage.googleapis.com.

Solução: múltiplos projetos + múltiplas keys + round-robin — cada key tem sua própria cota free tier (30 RPM, 1.500 RPD).

Keys Disponíveis ​

10 keys em 7 projetos diferentes:

ProjetoKeysDono
stone-history-499211-k94 keysbmbanho@gmail.com
gen-lang-client-00203008161 keybmbanho@gmail.com
gen-lang-client-00265789391 keybmbanho@gmail.com
gen-lang-client-03560736691 keybmbanho@gmail.com
gen-lang-client-03608426321 keybmbanho@gmail.com
gen-lang-client-03674202281 keybmbanho@gmail.com
gen-lang-client-07166879761 keybmbanho@gmail.com

Capacidade agregada (free tier): ~300 RPM / ~15.000 RPD (vs 30 RPM de uma key só).

Gerenciamento ​

bash
# Carregar o profile (se não tiver feito)
source ~/.config/gcloud/nexus-profile.sh

# Listar status de todas as keys
nexus::gemini::list

# Pegar a próxima key (round-robin automático)
nexus::gemini::next
# → AIzaSyDWGgW11M7-bANNggC1WAHBika8hXyixnQ

# Exportar como GEMINI_API_KEY (para scripts)
nexus::gemini::export
# → GEMINI_API_KEY setada

# Se uma key der 429 Rate Limit, marcar ela
nexus::gemini::ratelimit
# → Key fica em cooldown por 60s, round-robin pula ela

# Resetar o estado (volta todas a disponível)
nexus::gemini::reset

Uso para Agentes ​

Agentes que precisam chamar a Gemini API devem:

  1. Verificar se ~/.config/gcloud/nexus-gemini.py existe
  2. Chamar python3 ~/.config/gcloud/nexus-gemini.py next para obter a key
  3. Usar a key na chamada à API
  4. Em caso de erro 429, chamar nexus::gemini::ratelimit ou python3 ~/.config/gcloud/nexus-gemini.py ratelimit

Exemplo com curl:

bash
KEY=$(python3 ~/.config/gcloud/nexus-gemini.py next)
curl -s -X POST "https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent?key=$KEY" \
  -H "Content-Type: application/json" \
  -d '{"contents": [{"parts": [{"text": "Hello"}]}]}'

Exemplo com Python SDK:

python
import google.generativeai as genai
import subprocess

key = subprocess.check_output(
    ["python3", "/home/bruno/.config/gcloud/nexus-gemini.py", "next"]
).decode().strip()
genai.configure(api_key=key)

7. Segurança ​

  • A chave JSON está em ~/.config/gcloud/nexus-share-key.json — FORA do repositório NEXUS. Nunca mover para dentro do workspace.
  • Permissão do arquivo: 600 (só o dono lê).
  • A SA não tem permissões de faturamento, nem de gerenciamento de projeto, nem de criação de outras SAs — apenas recursos (storage, AI, BigQuery).
  • Para revogar a chave: gcloud iam service-accounts keys delete <key-id> --iam-account=nexus-share@...
  • Para rotacionar: deletar chave antiga, criar nova, atualizar o arquivo.
