Anchored Summary — Session 2026-06-25 ​

Bug: Qdrant Collection Vector Schema Mismatch ​

  • Problem: nexus_ghost and nexus_reflections collections had unnamed vectors; all code sends {"text_dense": vector}.
  • Root cause: Created during init without named vector. Only nexus_ast worked (manually created with correct schema).
  • Fix: Recreated both collections with text_dense (768d, Cosine).
  • Status: All 3 collections now consistent.

Fix: Ollama Container OOM ​

  • Problem: Ollama container (4G RAM limit) OOM-killed when loading both models.
  • Root cause: 6GB VRAM sufficient. Bottleneck was 4G container RAM — non-offloaded layers (~560MB) + nomic (~200MB) + Ollama overhead exceeded 4G.
  • Fix: docker-compose.standalone.yml → 4G → 8G.
  • GPU: 29 layers offloaded (GTX 1660S, 6GB VRAM). Embedding runs on CPU.

Bug: L2 Phase 2 Blocked by Concurrency Pile-up ​

  • Problem: Multiple concurrent run_l2_enrichment() (POST /enrich + watcher + background loop) fighting over 2-permit semaphore. Phase 1 never drained → Phase 2 never executed.
  • Evidence: 357 L2 rows in PG, 0 in Qdrant. No "phase 2" logs since restart.
  • Fix: TRUNCATE blobs CASCADE + Qdrant clear + restart. Background loops alone process cleanly.
  • Result: Phase 2 runs every cycle now.

Pipeline Performance (fresh ingest) ​

  • 266s uptime: 13 blobs ingested, 238 L0, 11 L1, 7 L2
  • GPU: 69-85% during L2 gen, 1780 MiB VRAM
  • RAM: ~5.3G/7.6G used, 2.3G available, 1.3G zram
  • Idle GPU pattern: L1 (CPU) outpaces L2 (GPU). Expected in dev.
